# Getting Started with Create React App

This project was bootstrapped with [Create React App](https://github.com/facebook/create-react-app).

## Backend Setup

The product catalog now uses MySQL instead of localStorage.

1. Create a MySQL database named `global_threads`, or run [database.sql](database.sql) to create it and the `products` table.
2. Copy [.env.example](.env.example) to `.env` and set `DB_HOST`, `DB_USER`, `DB_PASSWORD`, and `DB_NAME`.
3. Run `npm install`.
4. Start the API with `npm run server`.
5. Start the React app with `npm start`.

Product images are stored in MySQL as image data, and deleting a product from the UI removes it from the database too.

## Spring Boot Backend (Alternative)

If your requirement is Spring Boot, use the backend in [spring-backend](spring-backend):

1. Keep your existing `.env` values (`DB_HOST`, `DB_USER`, `DB_PASSWORD`, `DB_NAME`, `BACKEND_PORT`).
2. Open a terminal in [spring-backend](spring-backend) and run `mvn spring-boot:run`.
3. Keep the React app running with `npm start` from the project root.

The Spring Boot API matches the same endpoints used by the frontend:

- `GET /api/health`
- `GET /api/products`
- `PUT /api/products/sync`
- `DELETE /api/products/{id}`
